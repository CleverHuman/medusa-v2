# 供应商申请与入职流程 - 交互设计

## 核心交互功能

### 1. 多步骤申请表单
- **步骤导航**: 顶部进度条显示当前步骤（基本信息 → 产品详情 → 位置信息 → 联系方式 → 审核状态）
- **分步验证**: 每步都有必填字段验证，防止跳步提交
- **数据保存**: 每步自动保存到本地存储，防止数据丢失
- **前进后退**: 允许用户在步骤间导航，保持已填写数据

### 2. 动态表单字段
- **市场经验**: 选择"是"时显示额外字段（市场名称、经验年限）
- **产品类别**: 多选复选框，选择后显示相关子类别
- **联系方式**: 动态添加多个联系选项（Telegram、Signal、Jabber、Email）
- **钱包地址**: 支持BTC、XMR、USDT多币种输入

### 3. 实时验证与反馈
- **PGP签名验证**: 实时检查PGP签名的格式正确性
- **钱包地址格式**: 验证各类加密货币地址格式
- **联系信息**: 验证邮箱、Telegram等格式
- **即时反馈**: 输入框旁显示验证状态图标

### 4. 申请状态跟踪
- **提交确认**: 显示"申请已提交"动画确认
- **状态面板**: 显示当前审核状态（待审核 → 审核中 → 需要面试 → 已批准/被拒绝）
- **进度更新**: 模拟审核进度，包括产品范围评估、经验验证等
- **通知系统**: 模拟邮件/消息通知功能

### 5. 管理员审核界面
- **申请列表**: 显示所有待审核供应商申请
- **筛选功能**: 按产品类别、经验、状态筛选
- **详情查看**: 点击查看完整申请信息
- **审核操作**: 批准、拒绝、要求面试等操作按钮

### 6. 响应式设计
- **移动端优化**: 表单在移动设备上自适应布局
- **触摸友好**: 大按钮和易于点击的表单元素
- **键盘导航**: 支持Tab键导航所有表单元素

## 用户体验流程

1. **首次访问**: 显示申请入口和流程说明
2. **开始申请**: 进入多步骤表单，显示进度
3. **填写信息**: 分步骤填写，实时验证
4. **提交申请**: 确认信息并提交
5. **等待审核**: 查看申请状态和预计处理时间
6. **审核结果**: 接收通知并查看结果

## 技术实现要点

- 使用本地存储保存表单进度
- 实现平滑的步骤切换动画
- 添加加载状态和错误处理
- 支持表单数据的导入导出
- 提供打印友好的申请确认页面
